Featured Project: S&P 500 Risk ESG Analysis with AI & Machine Learning

IN PROGRESS

This intensive project leverages Python, SQL, and Tableau to construct and analyze ESG scores for S&P 500 companies, structured into 3 main phases that progressively increase automation, depth, and decision value.


Phase 1: Integrated existing ESG datasets by scraping preset scores from Yahoo Finance (Sustainalytics) and MSCI. Established a clean SQL database foundation with company metadata, sector/industry classifications, and baseline E/S/G pillar scores.


Phase 2: Built an automated AI/ML web scraping pipeline to extract ESG-related KPIs and controversy events from corporate filings, sustainability reports, and news. Applied natural language processing to map disclosures to MSCI’s 33 key issues, and implemented machine learning models in Python to compute Sustainalytics-style risk metrics (exposure, management, unmanaged risk) with configurable materiality weights at the sector, industry, and sub-industry levels.


Phase 3: Developed interactive Tableau dashboards to translate ESG risk into actionable insights. Visualizations include peer benchmarking, sector-level heatmaps, time-series trendlines, and drill-downs into material issues and controversies—allowing stakeholders to effectively evaluate ESG risks, identify high-exposure firms, and support strategic decision-making.

Stuff+/wStuff+ Model

An R-based machine learning pipeline that estimates Stuff+ and wStuff+ from raw pitch-tracking data (TrackMan and/or TruMedia data). The system trains XGBoost models with pitcher-aware cross-validation and calibrates outputs to a standardized 100-based scale. Without exposing specific inputs, model explainability validates baseball-relevant relationships and highlights where a pitcher's repertoire over/under-performs expectations.

Final deliverables include pitch and pitcher-level leaderboards, usage-weighted (wStuff+) summaries, and contextual splits to support pitch design and game planning.

xGB% Model

An R-based machine learning pipeline that predicts xGB using select ball-flight characteristics. The model leverages XGBoost with grouped cross-validation and probability calibration, outputting tuning metrics including ROC-AUC and PR-AUC. Feature importance and SHAP analysis highlight which pitch traits most strongly influence groundball outcomes, while the final leaderboard tables compare actual vs. expected groundball rates across handedness matchups, and the metrics associated with them.

The model's output delivers actionable insights off precidtors, correlations, and deltas on how mechanics impact batted-ball tendencies.

MLB Interactive Plate Discipline Analysis

IN PROGRESS

A Python-focused codebase that scrapes, manipulates, and formats baseball data with the goal to visualize offensive performance in the strikezone. This visual details hot/cold zones of the hitter, their actual and expected statistics, performances vs specific pitch types, splits righty-lefty and lefty/right, and much more.


This project utilized Pandas for robust data cleaning, transformation, and aggregation of Statcast data, including dynamic zone assignment based on pitch location. Engineered a custom strike zone mapping function to classify pitch locations into 9 standard zones and out-of-zone regions using conditional logic and coordinate binning. Built a Dash web application with interactive components, including a dropdown menu and responsive Plotly visualizations based on user input. Designed custom Plotly strike zone visualizations, integrating statistical overlays (AVG and xAVG) and dynamically adjusting label placement for clarity and comparison. Implemented real-time callback functions to update figures based on player selection, enabling exploratory analysis of offensive performance by zone.
